# 普惠金融风险动态评估平台指标口径与生成原理详细说明

## 一、文档用途说明

本文档不是平台功能介绍，也不是产品说明书，而是专门回答“平台里的处理后数据是怎么得出来的”。重点解释四个问题：

1. 原始数据来自哪里；
2. 原始数据进入平台后经过了哪些清洗和结构化处理；
3. 首页六个核心指标、雷达图、区域画像、政策模拟结果分别是如何计算或映射出来的；
4. 当前哪些结果属于真实统计数据直接驱动，哪些结果属于基于真实样本的规则建模与推演。

因此，这份文档更适合作为“技术与方法说明”发给项目成员，用来回答“这些指标是怎么来的、为什么这样算、每一步处理的依据是什么”。

## 二、原始数据层：平台到底接入了什么

平台当前已经真实接入的数据底座，核心是广东省官方公开统计样本库。后端数据文件位置为：

- `assets/data/guangdong_official_raw_data.json`

从代码结构上看，后端通过 `backend/data_provider.py` 中的 `_load_guangdong_raw_data()` 加载这一底表。底表加载后被统一组织为两个层次：

- `metadata`：记录数据集名称、所属省份、版本号、采集时间等元信息；
- `records`：记录每一条结构化统计数据。

根据后端检索和展示逻辑，单条原始记录至少包含以下字段：

- `province`：省份
- `city`：城市，可为空
- `industry`：行业
- `indicator`：指标名称
- `value`：指标值
- `unit`：单位
- `period`：时间周期，如“2025年1-6月”“2024年全年”
- `frequency`：频率类型，如年度、半年、月度累计
- `yoy_pct`：同比增速
- `category`：指标类别
- `source_org`：来源机构
- `source_title`：来源标题
- `source_url`：官方来源链接
- `notes`：补充说明

这意味着平台当前的数据不是“图片里的数字”，也不是“网页上手填的结果”，而是已经整理成统一 JSON 结构的官方样本底表。

## 三、原始数据进入平台后的处理流程

平台对原始数据的处理不是一次性的，而是分四层完成：字段标准化、筛选、排序、指标合成。

### 1. 字段标准化

后端首先对城市和行业做标准化处理，避免不同写法导致查询失败。

在 `data_provider.py` 中，城市通过 `CITY_KEYWORDS` 做归一：

- 广州 -> 广州市
- 深圳 -> 深圳市
- 佛山 -> 佛山市
- 东莞 -> 东莞市
- 珠海 -> 珠海市

行业通过 `MAIN_INDUSTRY_OPTION_MAP` 做主行业映射。例如：

- “制造业”不仅匹配“制造业”，还匹配“先进制造业”“高技术制造业”“食品制造业”“汽车制造业”等子行业；
- “软件和信息技术服务业”会映射“信息传输、软件和信息技术服务业”“科学研究和技术服务业”等。

这一步的作用是把用户在前端看到的主行业选项，与底表里的细分类行业统一起来。

### 2. 条件筛选

原始数据检索和首页指标刷新，都先经过 `_filter_records()` 这一步。它按四个条件过滤：

- 省份 `province`
- 城市 `city`
- 行业 `industry`
- 年份 `year`

年份筛选逻辑为：

- `all`：不过滤年份；
- `2026` / `2025` / `2024`：仅保留 `period` 中包含对应年份的记录；
- `older`：保留 2023 年及以前的记录。

这一步保证前端“省份、城市、行业、年份”四个筛选器是真正作用于底表，而不是只改标题不改数据。

### 3. 原始数据优先级排序

原始数据检索模块不仅做筛选，还做了优先级排序。排序逻辑在 `search_raw_data()` 中完成，主要包含两部分。

第一部分是匹配得分：

- 关键词命中省份、城市、行业、指标名称会加分；
- 有城市级字段会额外加分；
- 有真实数值会加基础分。

第二部分是业务优先级加权：

- “地区生产总值”优先级最高；
- “增加值”“社会消费品零售总额”“营业收入”“流动资产合计”“用电量”“存贷款余额”等指标依次加分；
- 主行业数据和城市级数据也会额外加分。

这说明平台不是把所有原始记录平铺给用户，而是优先展示“更重要、信息量更高、披露更完整”的记录。

### 4. 空值剔除

在 `search_raw_data()` 中，若 `value` 为 `None`、空字符串或“未披露”，则该条记录不进入展示结果。也就是说，前端原始数据卡片默认优先展示“有实际值”的记录，而不是让用户先看到一堆未披露项。

## 四、首页六个核心指标是如何计算出来的

这一部分是最核心的，因为队长问的就是“比如这六个核心指标到底怎么来的”。

首页六个指标并不是每个都直接对应一条统计记录，而是由若干条真实数据记录经过代理变量映射和加权合成得到。相关逻辑在：

- `backend/data_provider.py`
- `_build_real_dashboard_overview()`

当用户在首页选择了省份、城市、行业或年份后，平台就会调用这个函数，用真实样本重新计算一组 KPI。

### 1. 参与计算的底层原始记录

函数内部首先从筛选后的记录中挑选“最能代表当前场景”的关键指标：

- `gdp_record`：地区生产总值
- `social_retail_record`：社会消费品零售总额
- `deposit_record`：存款余额
- `loan_record`：贷款余额
- `revenue_record`：营业收入
- `current_assets_record`：流动资产合计
- `power_record`：用电量
- `sales_output_record`：规模以上工业销售产值
- `added_value_record`：增加值

这些记录不是随便取一条，而是用 `_pick_best_record()` 从候选记录中选出最优样本。排序依据主要是：

- 时间越新越优先；
- 同比信息越完整越优先；
- 数值越完整越优先。

### 2. 六个核心指标的计算逻辑

#### 指标一：区域经济景气度

公式：

`economy_score = clamp(72 + gdp_yoy * 3.2 + retail_yoy * 1.6)`

其中：

- `gdp_yoy` 是地区生产总值同比增速；
- `retail_yoy` 是社会消费品零售总额同比增速；
- `72` 是基础景气底分。

解释：

该指标用 GDP 和消费恢复共同刻画区域景气。GDP 代表宏观产出，社零代表消费端修复。GDP 的权重高于消费，因此乘数更大。

#### 指标二：行业发展健康指数

公式：

`industry_score = clamp(68 + sales_yoy * 1.7 + revenue_yoy * 1.2 + power_yoy * 0.9 + added_yoy * 1.1)`

其中：

- `sales_yoy`：规模以上工业销售产值同比；
- `revenue_yoy`：营业收入同比；
- `power_yoy`：用电量同比；
- `added_yoy`：增加值同比；
- `68`：行业健康基础分。

解释：

这一指标的设计逻辑是“产出 + 营收 + 开工活跃度 + 增长质量”四项联动。销售产值权重最高，因为它最直接反映订单和出货强度；营业收入次之；用电量作为生产活跃度代理；增加值反映产业附加价值修复。

#### 指标三：普惠金融环境指数

公式：

`finance_score = clamp(70 + deposit_yoy * 1.3 + loan_yoy * 1.0 + max(0, retail_yoy) * 0.8)`

其中：

- `deposit_yoy`：存款余额同比；
- `loan_yoy`：贷款余额同比；
- `retail_yoy`：消费恢复；
- `70`：金融环境基础分。

解释：

该指标并不是直接拿某个“金融环境”官方分数，而是用存贷款增长和消费修复共同代理金融环境。存款体现资金沉淀能力，贷款体现信用供给能力，消费恢复体现资金流通活跃度。

#### 指标四：小微企业经营健康

公式：

`micro_score = clamp(64 + revenue_yoy * 1.4 + assets_yoy * 0.8 - max(0, -sales_yoy) * 0.6)`

其中：

- `revenue_yoy`：营业收入同比；
- `assets_yoy`：流动资产同比；
- `sales_yoy`：销售产值同比；
- `64`：小微经营基础分。

解释：

营业收入上升说明企业经营恢复较好；流动资产上升说明短期周转能力较强；若销售产值同比为负，则说明订单端走弱，需要对小微经营健康做扣分修正。

#### 指标五：普惠小微贷款不良率

公式：

`npl_value = clamp(3.15 - finance_score * 0.011 - micro_score * 0.008 + max(0, -revenue_yoy) * 0.03, 0.9, 3.8)`

其中：

- `finance_score` 越高，不良率越低；
- `micro_score` 越高，不良率越低；
- 若营业收入同比为负，则不良率会上升；
- 最终结果限制在 0.9 到 3.8 之间。

解释：

这个值在当前“真实数据筛选”模式下是代理测算值，不是直接从底表中抽取一条“不良率”记录。其业务逻辑是：金融环境强、企业经营健康，则不良率应下降；反之则上升。

#### 指标六：政策实施有效性

公式：

`policy_score = clamp((economy_score * 0.22) + (industry_score * 0.26) + (finance_score * 0.22) + (micro_score * 0.18) + ((4 - npl_value) * 18) * 0.12)`

解释：

该指标是综合评分，不是单独一条原始数据。它同时吸收了经济、行业、金融、小微和不良率五个维度的信息。其中行业健康权重最高，说明平台认为政策有效性的首要体现是行业修复；不良率则通过反向转换后纳入综合评分。

### 3. 趋势箭头是怎么来的

六个核心指标右侧的涨跌箭头，不是拍脑袋写的，而是由 `_fmt_trend()` 根据同比值自动生成。

规则如下：

- 若同比值为正，则显示 `+x.x%`，方向为 `up`
- 若同比值为负，则显示 `-x.x%`，方向为 `down`

其中不良率做了单独处理：因为不良率下降是好事，所以首页显示为负值趋势，并固定标记为 `down`。

## 五、雷达图是如何生成的

雷达图并不是独立计算一套新数据，而是对首页核心指标做二次映射，逻辑位于 `_build_real_dashboard_overview()` 返回的 `radar_data` 字段中。

雷达图六个维度为：

- 经济景气 = `economy_score`
- 行业健康 = `industry_score`
- 普惠环境 = `finance_score`
- 小微经营 = `micro_score`
- 信用环境指数 = `radar_credit`
- 政策效力 = `policy_score`

其中 `radar_credit` 的公式为：

`radar_credit = finance_score * 0.58 + micro_score * 0.42`

解释：

信用环境并不完全等于金融环境，它还要考虑小微企业自身经营状态，所以平台把金融环境和小微经营两项做了加权融合。

## 六、原始数据检索结果是如何变成展示卡片的

前端原始数据卡片显示的每一项内容都来自底表字段，而不是额外计算出来的假值。

单张卡片通常由以下字段构成：

- 左上来源机构：`source_org`
- 右上位置和时间：`province/city + period`
- 中间标题：`industry + indicator`
- 核心大数字：`value + unit`
- 标签区：`category + frequency + yoy_pct`
- 底部描述：`source_title`
- 链接：`source_url`

因此原始数据检索模块的本质是：把后端结构化记录重新渲染成可视卡片，并按优先级排序显示。

## 七、地图和省份侧栏里的处理后数据是如何得出来的

### 1. 全国热力图

全国热力图对应后端 `get_risk_map()`。

这里的 `map_data` 目前是省级数值列表，例如：

- 广东 95
- 上海 92
- 浙江 90
- 江苏 88

这部分当前属于“演示型建模展示数据”，不是从全国全量实时底表直接聚合得出。它用于表现区域风险差异和地图交互能力。

### 2. 地图脉冲散点

地图上广东、上海、北京等闪烁脉冲点来自 `scatter_alerts`。每个点包含：

- 名称
- 经纬度和脉冲权重
- 一句告警解释

这些数据的作用是表达“长期指数较优，但短期链路风险可能上升”的业务含义。

### 3. 省份侧栏数据

点击地图后右侧侧栏调用 `get_province_detail()`。这部分数据来源于后端内置的省份画像样本：

- `_PROVINCE_DATA`
- `_INDUSTRY_PIE`
- `_RISK_FACTORS`

例如广东省对应：

- `npl = 1.85`
- `credit = 95`
- `micro = 82`

同时配套：

- 行业分布饼图
- 风险因子柱状图
- 文本预警

这一部分当前是“省份画像样本 + 规则映射”的展示层，不是广东原始底表直接推导出的实时结果。它的作用是支撑地图下钻叙事和区域画像展示。

## 八、趋势预测图是如何得出来的

趋势图由 `get_trends()` 返回。当前主要展示不良率趋势。

实际序列为：

- `actual = [2.50, 2.45, 2.40, ..., 2.14]`

预测序列为：

- `prediction = [None]*10 + [2.15, 2.14, 2.12, 2.05, 1.95]`

置信区间上界和下界也单独给出。

模型标签在接口中明确写为：

- `LSTM (128 units, 2 layers, DropOut=0.2)`
- `confidence_level = 95.8%`

因此，趋势图当前属于“历史演示序列 + LSTM 预测结果”模块，其中算法说明是真实写入系统的，数列本身属于演示型时间序列样本。

## 九、政策模拟结果是如何得出来的

政策模拟的计算逻辑位于 `_build_policy_result()`，这是平台里最完整的一段“处理后结果生成链路”。

### 1. 输入变量

用户在前端调整三个变量：

- 政府担保拨备率 `guarantee_rate`
- 利率优惠幅度 `interest_offset`
- 补贴覆盖等级 `subsidy_level`

### 2. DID 净政策效应

平台当前用一个简化的政策净效应公式表示：

`did_effect = (guarantee_rate - 10) * 0.018 + abs(interest_offset) * 0.42 + subsidy_level * 0.12`

解释：

- 担保率高于 10% 后开始产生额外增量效应；
- 利率优惠幅度越大，短期缓释作用越强；
- 补贴等级越高，产业端修复作用越强。

这不是正式计量论文意义上的真实估计式，而是对 PSM-DID 逻辑的简化模拟表达，用于竞赛阶段的情景推演。

### 3. 预测不良率

当前基准不良率设为：

`current_npl = 2.14`

政策实施后的新不良率为：

`new_npl = max(0.8, current_npl - did_effect)`

解释：

政策净效应越强，不良率越低；但为了保证结果符合金融常识，最低下限被限制为 0.8。

### 4. 财政成本

财政成本公式为：

`fiscal_cost = guarantee_rate * 0.8 + subsidy_level * 2.5`

单位为亿元。

解释：

担保拨备率和补贴等级分别对应财政成本的两个主要来源。

### 5. 受益企业数

公式：

`benefit_enterprises = int(12000 + did_effect * 8000)`

解释：

政策效应越强，理论上覆盖到的企业数量越多，因此平台用线性放大方式近似这一覆盖面效果。

### 6. ROI

平台当前统一口径为：

`roi = benefit_enterprises / (fiscal_cost + 0.1)`

口径名称为：

- 每亿元财政投入收益企业数

这意味着 ROI 不是百分比，也不是收益率，而是“每投入 1 亿元财政资金，理论上可受益多少家企业”。

### 7. 预测路径

为了在图上展示政策实施后的不良率下行过程，平台生成三个月路径：

`step_reduction = (current_npl - new_npl) / 3`

随后依次生成三个月值：

- `current_npl - step_reduction`
- `current_npl - 2 * step_reduction`
- `new_npl`

### 8. 风险预警

当参数过强时，平台会触发风险提示，例如：

- 担保拨备率过高，提示地方财政流动性压力；
- 利率降幅过大，提示银行净息差压缩；
- 补贴等级过高，提示需与专项债协同。

### 9. 评分卡

评分卡包括四项：

- 效率分 `efficiency_score = min(100, roi / 200)`
- 覆盖分 `coverage_score = min(100, benefit_enterprises / 220)`
- 实施分 `implementation_score = max(55, 92 - max(0, guarantee_rate - 18) * 1.5 - max(0, subsidy_level - 2) * 4)`
- 综合分 `composite_score = efficiency * 0.4 + coverage * 0.35 + implementation * 0.25`

这说明政策模拟器最终不是只输出一个 DID 值，而是输出一整套“政策效果 + 成本 + 覆盖 + 可实施性”的综合评价结果。

## 十、行业演化视图和产业链图谱的数据属于哪一类

这两部分当前更适合表述为“真实样本驱动的风险建模展示层”。

### 1. 行业演化视图

行业演化视图中的：

- 风险等级
- 风险分值
- 主压力量纲
- 结论建议
- 节点热度排序

目前主要来自前端维护的行业场景样本库和后端行业图谱结构，目的是把“行业风险随时间和链路变化”的逻辑表达出来，而不是直接作为统计局原表的逐字段翻译。

### 2. 产业链图谱

产业链图谱在 `get_industry_graph()` 中定义。每个节点都有：

- `risk_level`
- `risk_score`
- `description`
- `policy_focus`
- `recommended_adjustments`

这里的 `recommended_adjustments` 直接和政策模拟器联动，例如：

- `guarantee_rate_delta`
- `interest_offset_delta`
- `subsidy_level_delta`

因此图谱模块更接近“策略联动层”，不是单纯数据展示层。

## 十一、哪些处理后结果是真实数据直接驱动，哪些是代理测算或建模结果

为了保证表述真实，必须把三类结果分清楚。

### 1. 真实数据直接驱动

- 原始数据检索卡片

## 十二、可以直接对外的结论性表达

如果要对外简洁表达本平台的指标生成逻辑，可以概括为：

平台首先接入广东省及部分城市的官方公开统计样本，将原始数据结构化为统一底表；然后通过省份、城市、行业和年份四维筛选，选出与当前场景最相关的 GDP、社零、存贷款、营业收入、流动资产、用电量、销售产值和增加值等真实记录；再利用统一的标准化、方向修正和加权规则，生成首页六大核心指标和雷达图；最后进一步通过区域建模、行业演化、链路传导、LSTM 趋势预测和基于 PSM-DID 逻辑的政策沙盘，形成从原始数据到风险识别、再到政策建议的完整闭环。

换言之，平台中的“处理后数据”不是凭空生成，而是按照“真实样本 -> 结构化筛选 -> 代理变量映射 -> 评分合成 -> 模型推演”的链路逐层得出。
