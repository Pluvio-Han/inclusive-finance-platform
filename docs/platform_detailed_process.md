# 区域与行业大数据驱动的普惠金融风险动态评估平台详细运行过程说明

## 一、平台定位与总体目标

本平台名称为“区域与行业大数据驱动的普惠金融风险动态评估平台”。平台的核心目标不是做单一的数据可视化页面，而是构建一个完整的闭环式决策支持流程：先从真实统计口径中抽取底层数据，完成清洗、标准化和结构化处理；再基于区域、行业和企业经营逻辑构建指标体系；随后通过风险监测、产业链分析和政策模拟三个模块进行联动分析；最后把分析结果沉淀为可展示、可解释、可导出的结构化决策建议报告，用于“三创赛”路演展示和答辩说明。

从整体架构上看，平台采用的是 FastAPI 后端加原生 JavaScript 前端的轻量化方案。后端负责数据管理、指标计算、政策模拟、报表接口和静态资源分发；前端负责交互控制、图表渲染、地图下钻、筛选查询和报告导出。之所以选择这一套架构，是因为它足够稳定、足够可控，而且便于在无网环境下完整演示。当前系统已经做到前端资源、地图 GeoJSON、图标字体、ECharts 依赖全部本地化，满足离线演示要求。

## 二、平台使用的数据类型与来源

平台当前的数据底座分成两层。

### 1. 真实接入的数据底座

这一层已经接入到平台并真实参与展示与计算。目前真实接入的重点样本是广东省统计口径数据，包含省级和部分市级样本。真实数据主要来自广东省统计局等官方公开统计信息，覆盖了地区生产总值、规模以上工业营业收入、流动资产合计、用电量、部分行业累计值、同比增速等结构化指标。平台中“原始数据检索”模块和首页六大 KPI 的主要输入，就来自这一层真实样本。

### 2. 基于真实样本的建模展示数据

这一层主要用于全国热力图、行业演化视图、产业链风险传导图谱、区域下钻规划和政策沙盘中的部分可视化内容。原因在于：目前真实底表已经能够支撑广东省及部分城市的结构化展示，但还不足以直接支撑全国所有省份、所有行业、所有链路节点的完整真实覆盖。因此平台在建模展示层采用了“真实样本 + 规则映射 + 风险建模”的方式，把广东真实样本中的统计特征、经济含义和风险结构映射到更大范围的演示场景中。

也就是说，平台没有把所有图表都伪装成“全国全量真实数据”，而是以广东真实样本为基础，向区域画像、行业画像和风险链路做了可解释的扩展建模。

这一点在答辩时可以明确表述为：平台当前已经完成“真实数据接入 + 模型展示联动”的第一阶段，真实接入重点放在广东省，其他区域和行业模块则是在真实统计样本基础上进行结构化推演和可视化建模，目的是验证方法链路和平台能力，后续可以按相同流程扩展到更多省市和行业。

## 三、数据接入方式

平台当前的数据接入方式，不是直接对接某个在线数据库，而是采用“离线样本入库 + 结构化 JSON 底表管理”的方式。这样设计主要有两个原因：

1. 竞赛演示环境强调稳定性和可控性。若依赖实时在线接口，一旦网络不稳定、接口限流或证书异常，现场演示风险较高。
2. 官方统计数据本身具有发布周期和口径要求，很多时候适合按批次采集、整理、清洗后再统一接入，而不是每次前端请求时临时抓取。

目前平台的数据文件已经被整理为结构化底表。每一条记录至少包含以下字段：省份、城市、行业、指标名称、指标值、单位、时间周期、频率类型、同比信息、来源机构、来源标题和来源链接。这样做的好处是，前端可以按“省份-城市-行业-年份”四维筛选直接检索底表，后端也可以基于这些字段快速完成指标汇总和解释性输出。

换句话说，平台的数据接入不是“页面直接去网页抓数据”，而是先把官方公开数据沉淀为标准结构，再由后端统一管理。这种方式更符合正式系统的工程逻辑，也方便后续扩展到更多区域。

## 四、数据清洗与预处理流程

真实数据接进来之后，平台并不会原样直接展示，而是经过多步处理。

### 1. 口径统一

不同来源的数据单位可能不同，例如“亿元”“万元”“亿千瓦时”“%”等，平台会在入库时保留原单位，并在展示和聚合时做标准化处理，避免同类指标混用不同计量尺度。平台不会把所有值都强行转换成一个单位，而是保留“原始统计口径 + 前端格式化展示”的双层机制，这样既便于追溯，又便于展示。

### 2. 维度规范化

地区名称、城市名称、行业名称在不同来源中可能存在不同写法。平台会做行政区划名称清洗，例如“广东省”统一为“广东”，“广州市”统一为“广州市”，同时前端也做了名称标准化处理，保证地图点击名称和后端查询字段一致，避免前后端因名称不一致导致下钻失败。

### 3. 缺失值和异常值处理

对于未披露、未统计或不适合直接展示的字段，平台不会用假值硬填，而是保留“未披露”语义，并在搜索结果中优先显示已披露、可解释、较高优先级的数据。平台已经对检索结果做过排序优化：对于“未披露”项，优先不展示或降低展示优先级，避免用户看到大量无效卡片。这一处理符合真实统计口径的严谨性。

### 4. 时间口径统一

不同指标可能是月度累计、半年累计、年度值或者阶段值。平台在底表中保留 `period` 和 `frequency` 字段，并在筛选时允许按年份过滤。前端不会简单把不同时间口径的指标混成同一层逻辑，而是会在卡片上明确标出“2025年1-6月”“2024年全年”“2026年1-2月”等时间范围。

### 5. 结构映射

为了支撑首页 KPI、风险画像和政策沙盘，平台会把底层的多个真实指标映射成更高一层的综合指标。例如区域经济景气度并不是单条 GDP 记录，而是对 GDP、工业产值、消费、能源等宏观经济变量进行加权组合后形成的结果。行业健康、小微经营、普惠环境等也是如此，它们都不是“直接抄表”，而是“基于官方统计数据和业务规则计算得到的合成指标”。

## 五、平台采用的核心技术与算法

平台的技术可以分成四类：工程技术、指标技术、预测技术和因果评估技术。

### 1. 工程技术

后端使用 FastAPI 作为接口层，通过 Pydantic 做请求和响应的 Schema 校验，确保前后端通信结构稳定。静态资源路由做了路径安全校验，避免路径穿越问题。前端使用原生 JavaScript 管理页面状态，用 ECharts 实现雷达图、地图、折线图、柱状图、热力图和关系图，整体不依赖外部 CDN。报告导出使用浏览器内 iframe 打印方案生成结构化报告，而不是简单截图。

### 2. 指标技术

平台围绕区域环境、行业健康、小微经营、信用环境、政策效力等维度构建了一套多指标合成框架。每个综合指标都对应若干底层统计量，再通过加权、标准化和方向一致性修正形成统一得分。比如有的指标越高越好，有的指标越低越好，例如不良率就是负向指标，处理时要先做方向转换，才能纳入统一评分体系。

### 3. 预测技术

平台在不良率趋势模块中采用的是 LSTM，即长短期记忆网络，用于对普惠小微贷款不良率做时间序列趋势预测。LSTM 的作用不是“凭空生成未来”，而是在已有历史序列的基础上学习时间依赖关系，输出后续若干期的趋势值。平台同时给出了置信区间，用于表示预测的不确定性范围。系统打磨过程中还专门修正过置信区间下界逻辑，避免出现不符合金融常识的预测结果。

### 4. 因果评估技术

政策模拟器的核心逻辑建立在 PSM-DID 的方法框架上。PSM-DID 可以理解为“先匹配、再双重差分”：先尽量让接受政策支持和未接受政策支持的样本在初始条件上更可比，然后再比较政策实施前后的差异变化，从而估计政策净效应。平台当前出于竞赛阶段和数据覆盖范围的限制，还没有做到真实企业微观面板级别的全量 PSM-DID 训练，但在方法论口径上已经完全按这一框架组织，输出的 DID 估计量、政策效力和 ROI 都是围绕这一因果评估逻辑构造的。

## 六、平台主要模块的运行逻辑

### 1. 综合驾驶舱

首页的作用是“总览当前风险状态”。它不是一个静态首页，而是整个平台的数据总入口。用户当前可以通过省份、城市、行业和年份对原始数据做筛选，然后平台根据筛选结果动态刷新页面上方的六大 KPI 和雷达图。也就是说，这六个框并不是写死的演示数字，而是可以随着真实样本的筛选条件变化而变化。

当前六个核心指标分别是：

- 区域经济景气度
- 行业发展健康指数
- 普惠金融环境指数
- 小微企业经营健康
- 普惠小微贷款不良率
- 政策实施有效性

这六个指标属于平台第一层结构化风险判断，它们背后分别映射真实底表中的若干指标，再通过标准化、方向统一和权重计算得到。

### 2. 原始数据检索模块

原始数据检索模块承担了“让真实数据可见”的功能。用户可以按省份、城市、行业和年份筛选底层数据，平台会返回匹配到的官方原始记录，包括指标名称、指标值、时间口径、同比信息和官方来源链接。这样一来，平台就不仅能展示处理后的结果，还能向评委证明“这些结果不是空算出来的，背后有真实统计数据支撑”。

这部分非常重要，因为它增强了平台的可信度。以前如果评委问“你这些数字从哪来”，只能口头解释；现在可以直接现场打开原始数据检索模块，选择广东、选择广州、选择制造业、选择年份，然后展示真实来源和原始值，形成“原始数据 -> 指标处理 -> 风险判断”的证据链。

### 3. 全国热力图与省份侧栏

全国热力图模块展示的是普惠金融环境指数的区域分布。当前地图并不是宣称“全国全量真实实时数据”，而是基于广东真实样本与区域建模逻辑形成的展示层。点击某个省份后，右侧会弹出省份画像侧栏，显示不良率、信用环境、小微健康，以及主要行业分布和风险因子贡献。这部分的作用是把“全国地图”下钻成“单省画像”。

这一模块的业务逻辑是：先通过地图呈现宏观空间差异，再通过侧栏展示单个省份的风险特征和结构分布，让评委看到平台具备“从宏观到局部”的分析能力。

### 4. 行业演化视图

行业演化视图用于展示行业风险如何在时间维度上演化，包括行业风险指数曲线、风险驱动因子热力拆解、重点节点风险排序和传导链路结论。它的核心不是给出一个静态行业评分，而是解释“为什么这个行业风险上升了，是账期、成本、补贴还是景气在推动变化”。

当前行业演化视图中，不同行业的部分展示内容是基于真实样本与风险逻辑建模生成的。也就是说，它是“真实样本驱动的结构化演化展示”，而不是完全无依据的虚构。

### 5. 区域下钻规划

区域下钻规划模块展示的是区域层面的政策落地建议。它会结合区域信用环境、小微经营健康、产业结构和风险因子，给出区域诊断摘要和政策建议组合。这里的逻辑是“看完全国分布之后，不是停留在看图，而是继续往政策落地走”。

### 6. 政策模拟平台

这是整个平台最强调“闭环”的部分。用户可以调整政府担保拨备率、利率优惠幅度、产业补贴覆盖等级等变量，系统会输出预测不良率、DID 估计量、受益企业数、财政成本和政策 ROI。这里的 ROI 已经统一为“每亿元财政投入收益企业数”，这一点是平台当前已经统一的业务口径。

更关键的是，平台不会只给一个数，而是会输出：

- 情景摘要
- 方案类型
- 政策工具组合
- 传导路径
- 适用对象
- 执行约束
- 决策证据
- 评分卡
- 风险预警
- 导出报告

也就是说，政策模拟平台的最终输出不是“结论数字”，而是“结构化决策卡片”。

## 七、平台如何从数据一步步得出结论

平台整体分析链路可以概括为六步：

1. 采集和整理真实统计数据，当前重点是广东省及部分城市的官方公开统计样本。
2. 对底层数据进行清洗和标准化处理，包括单位统一、时间口径统一、名称规范化、缺失值处理和结构字段整理。
3. 把原始统计数据映射为可计算的中层指标，例如经济景气、行业活跃度、融资环境、小微经营特征等。
4. 将中层指标通过统一权重逻辑组合成首页六大 KPI 和雷达图，从而形成第一层结构化风险判断。
5. 在行业演化、区域下钻和产业链图谱模块中，进一步把这些指标投射到“时间”“空间”“链路”三个维度，构建风险监测能力。
6. 在政策模拟平台中，把政策变量输入到基于 PSM-DID 逻辑的模拟框架里，估计政策净效应，再结合 LSTM 趋势结果和多维评分体系，输出最终决策建议和报告。

因此，这个平台不是“一个图一个图堆出来”，而是“数据底座 -> 指标计算 -> 风险识别 -> 情景模拟 -> 证据导出”的完整链路。

## 八、当前真实落地与扩展建模的边界说明

这一部分需要明确区分，越清楚越显得平台真实、专业。

### 1. 当前已经真实落地的部分

- 广东官方原始数据样本库
- 按省份、城市、行业、年份筛选的原始数据检索
- 首页六大 KPI 随筛选条件动态变化
- 首页雷达图随筛选结果动态变化
- 决策报告导出流程
- 整体前后端工程与离线部署能力

### 2. 当前属于“真实样本 + 规则映射 + 建模展示”的部分

- 全国热力图
- 区域下钻中的部分省份画像
- 行业演化视图
- 产业链风险传导图谱
- 部分政策模拟场景解释文案

### 3. 当前还可继续扩展的部分

- 更多省份和更多城市的真实数据接入
- 更多行业的真实统计底表覆盖
- 用企业级微观样本替代当前部分启发式参数
- 将 PSM-DID 从方法论框架进一步落到更完整的真实样本估计
- 将 LSTM 预测输入替换为更长周期的真实时间序列

这一表述方式的优势在于：既体现了平台已经具备真实数据能力，也不会把当前版本夸大为“全国全量真实数据系统”，更适合竞赛答辩场景。

## 九、平台答辩时的核心价值表达

如果用一句偏比赛路演的话概括，这个平台的价值不在于“做了几个图”，而在于实现了从真实统计数据到底层检索、从多维指标到风险识别、从产业链与区域画像到政策模拟、再到结构化报告导出的全流程闭环。平台一方面能回答“现在风险在哪里”，另一方面也能回答“为什么会有这个风险”，最后还能回答“如果要干预，应该怎么干预、成本多大、收益如何”。

换句话说，它不是一个静态展示平台，而是一个可解释、可下钻、可模拟、可导出的普惠金融风险动态评估与决策支持平台。

## 十、总结性表述

本平台当前已完成“真实数据接入 + 风险建模展示 + 政策模拟闭环”的第一阶段建设。真实数据以广东省官方统计样本为核心，支持按省份、城市、行业和年份进行原始数据检索，并驱动首页六大核心指标和雷达图动态变化。在此基础上，平台进一步构建了区域热力图、行业演化视图、产业链风险传导图谱和政策模拟沙盘，利用 LSTM 完成趋势预测，利用 PSM-DID 逻辑完成政策净效应评估，最终输出结构化决策建议和可导出的分析报告。整体实现了“原始数据可追溯、指标计算可解释、风险判断可联动、政策建议可展示”的完整运行链路。
